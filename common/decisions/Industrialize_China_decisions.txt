##### OVERVIEW

# This is a general overview of how the mechanics for the state INDUSTRIALIZATION decisions work.
# Essentially, the decisions are static in terms of the decision ID and the states they integrate. (Not spawned from target triggers like other big decision lists)
# That means there's a lot of them. Sorry! Otherwise this coring needs to be done by events, and this actually turns it into a game mechanic rather than a DH-style RNG fest.
# To gain access to the set of decisions, a given tag has to fulfill the criteria in the "allowed" condition for the decision category.
# This decision category is "kr_china_province_INDUSTRIALIZATION_category," located in common\decisions\categories.
# This unlocks the entire group of decisions to integrate provinces and foreign concessions as core states for that tag.
# These decisions are meant for tags that have aspirations to become the sole government of China, which includes the Qing/North Zhili, the Fengtian Government, and the KMT by default.
# Other tags could also be added too! You can make additional tags require specific focuses, ideas, governments, or country flags to be included on the whitelist. Shandong/Shangqing Tianguo is one notable example.
# The groupings of states are based on the rough borders of the historical provinces of China (under the late Qing and the Nationalists partially)
# While the conditions, effects and visibility triggers are all constant, the PP cost and the time taken to "integrate" the province varies based on 1) tag and 2) previous effects caused by events, decisions, or national focuses
# How is this done? VARIABLES

##### HOW THE STATE INDUSTRIALIZATION VARIABLES WORK

# Default variables for all INDUSTRIALIZATION costs and durations is included in each decision
# This is given in case a tag doesn't define it
# I grouped them into three PP cost "tiers" roughly dependent on size, with a special discounted set for the concessions.
# If you're going to have mechanics in the forms of events or  that fiddle around with INDUSTRIALIZATION costs and times for a given tag (like, say, my work for CHI), then they should have a defined set of variables in their history file (see CHI - China.txt in history/countries for reference)

political_actions = {
	CHN_enable_industrialization_program = {
		icon = generic_construction
		allowed = { is_chinese_tag = yes }
		available = {
			has_tech = construction4
			JAP = {
				OR = {
					NOT = { has_war_with = ROOT }
					has_capitulated = yes
					is_subject = yes
				}
			}
			all_other_country = {
				is_han_chinese_tag = yes
				OR = {
					exists = yes
					is_in_faction_with = ROOT
					is_subject_of = ROOT
				}
			}
		}
		visible = { has_country_flag = has_united_china }
		fire_only_once = yes
		ai_will_do = { factor = 425 }
		cost = 150
		complete_effect = {
			log = "[GetDateText]: [Root.GetName]: Decision CHN_enable_industrialization_program"
			unlock_decision_category_tooltip = CHN_industrialize_china_decisions
			set_country_flag = CHN_enable_industrialization_decisions
			random_owned_state = {
				limit = {
					free_building_slots = {
						building = industrial_complex
						size > 1
						include_locked = yes
					}
					is_core_of = ROOT
				}
				add_extra_state_shared_building_slots = 2
				add_building_construction = {
					type = industrial_complex
					level = 2
					instant_build = yes
				}
			}
		}
	}
}
